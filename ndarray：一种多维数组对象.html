ndarray：一种多维数组对象

发表于 2020-02-29 分类于 Python3

NumPy 最重要的一个特点就是其 N 维数组对象（即 ndarray），该对象是一个快速而灵活的大数据容器。你可以利用这种数组对整块数据执行一些数学运算，其语法跟标量元素之间的运算一样。

引入 NumPy：

  • 这个是引入 NumPy 的习惯用法
  • 也可以使用 from numpy import *，但不建议使用此种方式。这是因为：
    • NumPy 的命名空间很大，包含许多函数
    • 其中一些的名字与 Python 内置的函数重名（比如 min 和 max）

ndarray 理解：

  • ndarray 是一个通用的同构数据多维容器——也就是说其中的元素必须是相同类型的
  • 每个数组都有一个 shape（一个表示各维度大小的元素）和一个 dtype（一个用于说明数组数据类型的对象）

1.2 NumPy 的数据类型

dtype （数据类型）是一个特殊的对象，它含有 ndarray 将一块内存解释为特定数据类型所需的信息

创建 ndarray 时指定 dtype

NumPy 的数据类型

dtype 是 NumPy 灵活交互其它系统的源泉之一。多数情况下，它们直接映射到相应的机器表示，这使得“读写磁盘上的二进制数据流”以及“集成低级语言代码（如 C、Fortran）”等工作变得更加简单。数值型 dtype 的命名方式相同：一个类型名（如 float 或 int），后面跟一个用于表示各元素位长的数字。

标准的双精度浮点值（即 Python 中的 float 对象）需要占用 8 字节（即 64 位）。因此，该类型在 NumPy 中就记作 float64。

二、NumPy 数组的运算

数组很重要，因为它使你不用编写循环即可对数据执行批量运算，NumPy 用户称其为矢量化（vectiorization）。

大小相等的数组之间的任何算术运算都会将运算应用到元素级。（数据分析暂时只用到大小相等的数组之间的运算。不同大小的数组之间的运算叫做广播（broadcasting），无需深入理解）

  • 数组与数组运算也是将对应的元素做算术运算
  • 数组与标量的算术运算会将标量值传播到各个元素
  • 数组之间做比较会生成布尔值数组

三、索引和切片

3.1 基本的索引和切片

NumPy 数组的索引是一个内容丰富的主题，因为选取数据子集或单个元素的方式有很多。

3.1.1 一维数组

一维数组的索引和切片很简单，看起来和 Python 列表的索引与切片极其类似

当你将一个标量值赋值给一个切片时（如 arr[5:8]=12），该值会自动传播（也就说后面将会讲到的“广播”）到整个选区。跟列表最重要的区别在于，数组切片是原始数组的视图。这意味着数据不会被复制，视图上的任何修改都会直接反映到源数组上。

  • 数组切片是原始数组的视图，所有对切片进行的修改将直接反映到源数组上
  • 切片 [:] 会给数组中的所有值重新赋值
  • 如果想要得到的是数组切片的一份副本而非视图，就需要明确地进行复制操作，例如 arr_slice = arr[3:7].copy()

3.1.2 高维数组

对于一个二维数组，各索引位置上的元素不再是标量而是一维数组

如果需要索引单个元素，则有两种方式（两种方式是等价的）：

  • 先索引一维数组，再索引单个元素
  • 直接传入一个逗号分隔的索引列表来选取

类似地，对于多维数组，如果省略了后面的索引，则返回对象会是一个维度低一点的 ndarry（它含有高一级维度上的所有数据）。

示例1：对于一个 2x2x3 的数组，

  • arr3d[0] 是一个 2x3 的二维数组
  • arr3d[0, 1] 是一个 1x3 的一维数组
  • arr3d[0, 1, 0] 才是一个标量元素

所有的切片操作和一维数组相同，都是视图操作。
