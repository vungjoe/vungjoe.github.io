函数和代码复用

发表于 2019-05-13 分类于 Python3

一、函数的定义与使用

函数是一段代码的表示，

  • 函数是一段具有特定功能的、可重用的语句组
  • 函数是一种功能的抽象，表达特定的功能
  • 两个作用：降低编程难度 和 代码复用

函数的定义：

  • 函数定义时，所指定的参数是一种占位符
  • 函数定义后，如果不经过调用，不会被执行
  • 函数定义时，参数是输入、函数体是处理、结果是输出（IPO）

1.1 函数的调用

调用是运行函数代码的方式：

  • 调用时要给出实际参数
  • 实际参数替换定义中的参数
  • 函数调用后得到返回值

1.2 参数传递

可选参数：函数定义时可以为某些参数指定默认值（定义时 def f(n, m=2)，种就构成了可选参数。可选参数必须放在必选参数的后面，也就是说：

可变参数：函数定义时可以设计可变数量参数，即不确定参数的数量：在可变数量参数前面加星号 * 。

参数传递的两种方式：

  • 按位置传递：fact(1, 2 )
  • 按名称传递：fact(m=1, n=2)

1.3 函数的返回值

函数可以返回 0 个或多个结果：

  • 函数使用 return 保留字来传递返回值
  • 函数可以有返回值，也可以没有；可以有 return ，也可以没有
  • return 可以传递 0 个返回值，也可以传递任意多个返回值

1.4 局部变量和全局变量

局部变量和全局变量是不同变量

  • 局部变量是函数内部的占位符，与全局变量可能重名但并不相同
    • 如果变量是基本数据类型，局部变量和全局变量不相同

二、代码复用和函数递归

2.1 代码复用

把代码当成资源进行抽象

  • 代码资源化：程序代码是一种用来表达计算的“资源”
  • 代码抽象化：使用函数等方法对代码富裕更高级别的定义
  • 代码复用：同一份代码在需要时可以被重复使用

代码复用的两种形式：

  • 函数：将代码命名，在代码层面建立了初步抽象
  • 对象：属性和方法，在函数之上再次组织进行抽象
    • 属性： <a>.<b>
    • 方法：<a>.<b>()

2.2 模块化设计

模块化设计的基本思想是自顶向下、逐步分解、分而治之，即将一个较大的程序按照功能分隔成一些小模块，各模块相对独立、功能单一、结构清晰、接口简单。

  • 通过函数或对象封装将程序划分为模块及模块间的表达
  • 具体包括：主程序、子程序和子程序间关系
  • 分而治之：一种分而治之、分层抽象、体系化的设计思想

紧耦合 松耦合

  • 紧耦合：两个部分之间交流很多，无法独立存在
  • 松耦合：两个部分之间交流较少，可以独立存在
  • 模块内部紧耦合、模块之间松耦合

2.3 函数递归

函数递归（或者叫递归函数）：在函数定义中调用函数自身的方式，就是递归。

两个关键特征：

  • 链条：计算过程存在递归链条
  • 基例：存在一个或多个不需要再次递归的基例
